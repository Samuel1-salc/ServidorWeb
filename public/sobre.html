Servidor Web Multithread

Trabalho academico de Redes de Computadores — implementacao em Python com sockets TCP e threads

Python 3 Socket TCP HTTP/1.1 Threading Porta 8080

O que e o projeto?

Um servidor web funcional que recebe requisicoes de multiplos clientes ao mesmo tempo, interpreta o protocolo HTTP na camada de aplicacao e entrega paginas HTML armazenadas localmente. Toda a implementacao usa apenas a biblioteca padrao do Python (socket e threading), sem frameworks externos.

Funcionalidades implementadas

Comunicacao TCP Servidor escuta em 127.0.0.1:8080 e aceita conexoes via socket.
Protocolo HTTP Interpreta requisicoes GET /caminho HTTP/1.1 e monta respostas completas.
Multithread Cada cliente e atendido em uma thread separada, sem bloquear os demais.
Arquivos estaticos Paginas HTML sao lidas de public/ e enviadas com cabecalhos corretos.
Erro automatico Arquivos inexistentes e metodos invalidos exibem templates/404.html.
Testes Suite automatizada e cliente concorrente validam status, cabecalhos e corpo.

Respostas HTTP

Todas as respostas incluem cabecalhos padronizados para o cliente interpretar corretamente o conteudo:

HTTP/1.1 200 OK
Content-Type: text/html; charset=utf-8
Content-Length: [tamanho em bytes]
Connection: close
Situacao Status HTTP Comportamento
Arquivo encontrado em public/ 200 OK Envia o conteudo HTML da pagina solicitada
Arquivo inexistente ou rota bloqueada 404 Not Found Serve automaticamente templates/404.html
Metodo diferente de GET (ex: POST) 400 Bad Request Serve a pagina de erro com status 400

Relacao com Redes de Computadores

  • Camada de aplicacao: protocolo HTTP simplificado (metodo GET, status line, cabecalhos e corpo).
  • Camada de transporte: comunicacao confiavel via TCP com sockets como interface entre aplicacao e rede.
  • Concorrencia: threads permitem atendimento paralelo, simulando o comportamento de servidores reais.
  • Modelo cliente-servidor: o navegador ou scripts/client.py atuam como clientes; server.py centraliza o atendimento.

Como executar

1. Iniciar o servidor python server.py
2. Testes automatizados python tests/test_server.py
3. Cliente concorrente python scripts/client.py
